4 # Summary of functionality
5 
6 The Disk API is fairly big and provides:
7 
8 - Abstracted Path objects for files and folders
9 - Abstracted File object
10 - Temporary files and folders
11 - A better walk() function for getting files/subfolders inside a folder
12 - Remote disk objects that work over SSH
13 - __One unified API for all of the above__
14 - An improved ZipFile class
15 - A configuration file parser for modifying config files with key+value pairs
16 
17 # Detailed functionality
18 
19 ## Representing file and folder paths
20 
21  from Disk import Local
22  filePath = Local.FilePath("path/to/file.txt")
23  folderPath = Local.FolderPath("path/to/local/folder")
24 
25 `FilePath` instances have methods similar to what is available in python's built in `os.path`, except that the
26 first argument of all methods in `os.path` is not needed because functions in `os.path` are not attached to a
27 class instance, whereas functions belonging to a `FilePath` class are. (This is similar to the move from all
28 string functions being stored in the `string` module to becoming methods of string objects.) For example, to
29 get the `dirname` of a path, do:
30 
31  folderPath = filePath.dirname()
32 
33 `folderPath` will be an instance of `Disk.Local.FolderPath`.
34 
35 90% of functions available in `os.path` are identically implemented as object methods in FilePath and FolderPath.
36 In addition, some additional functions (many from `shutil`) are also available:
37 
38 - For `FilePath` objects:
39  - `move(destinationFilePath)`
40  - `rename(newFilename)`
41  - `copy(destinationFilePath, shouldCopyDates)`
42  - `create()` - Similar to `touch`
43 - For `FolderPath` objects:
44  - `copy(destinationFilePath)`
45  - `remove(isRecursive)`
46  - `mkdirs()`/`makedirs()` - Same as `os.path.makedirs`
47  - `walk(...)` - See below
48 
49 ### Walking a folder
50 
51 Python has 2 stock implementations of a folder walker, each with its own features. A 3rd implementation is
52 introduced here, with even more features:
53 
54  subItems = folderPath.walk(isRecursive, wantFiles, wantFolders, followSymlinks=False, nameGlob="*", topDown=True)
55 
56 `subItems` will be a `list` of `FilePath`s and/or `FolderPath`s, depending on `wantFiles` and `wantFolders`.
57 
58 ### Temporary folders
59 
60 Temporary folders are the same as a `Local.FolderPath` above, with a pre-set folder name and path:
61 
62  from Disk.TempFolder import TempFolderPath
63  folder = TempFolderPath()

65 ## File objects
66 
67 File objects here are similar to the objects returned by python's built in `open()` function, but have some
68 differences/improvements. Here are some examples:
69 
70  from Disk import Local
71  file_ = Local.File("/path/to/file", "r")
72 
73  if file_.isWritable():
74  # determines if the file can be written to using the mode specified
75  if file_.isClosed():
76  # do something
77  print(file_.getPath())
78 
79  with file_:
80  # use standard read/write methods here
81  with file_:
82  # files can be reopened later, after being closed by the `with` statement above

84 ### Temporary files
85 
86 Temporary files are the same as a `Local.File` above, except they have the option to be deleted when they are destructed.
87 `NamedSecureTempFile` is basically a combination of python's built-in `tempfile.NamedTemporaryFile` and `tempfile.mkstemp`.
88 
89  from Disk.TempFile import NamedSecureTempFile as TempFile
90  file_ = TempFile(delete=True)
91  with file_:
92  # do something
93  file_.destruct() # permanently closes the file, and possibly deletes it

95 ## Remote files and folders using SSH
96 
97 There are identical `FilePath`, `FolderPath`, and `File` classes as mentioned above, but for SSH! Use them the same way.
98 Also, these 3 classes can be mixed interchangably in arguments, so for example, a `Local.FilePath` can be copied to a
99 `SSH.FilePath` using the same API:
100 
101  from Disk import Local, SSH
102  source = Local.FilePath("path/to/file.txt")
103  dest = SSH.FilePath("remote/path/to/file.txt")
104  source.copy(dest)

106 ## Improved ZipFile
107 
108 The built in python `ZipFile` module/class has a bug where unix chmod file attributes within the zip are not restored
109 when unzipped. The new `ZipFile` module included here fixes that.
110 
111  from Disk.CompressedFile import ZipFile
112  zipFile = ZipFile("path/to/myZippedFile.zip")
113  zipFile.extract(folder)

115 ## Configuration file modifier
116 
117 This class has been designed to modify configuration files which have a list of key+value pairs in a certain format.
118 This format includes most configuration files.
119 
120  from Disk.ConfigFile import Parser_m1 as ConfigFile
121  ConfigFile.changeConfig(filePath, newLine, commentChar='#', assignChar=' ', sectionName=None, sectionMatch='[')
122 
123 `commentChar` is the character or string that is placed at the beginning of a comment in the config file
124 `assignChar` is the character or string between the key and value. Sometimes this is an equals sign.
125 `sectionName` should be used only if the config file has sections, ex. `[mysqld]`.
126 `sectionMatch` is the beginning character/string before the `sectionName`. This is ignored if `sectionName` is `None`.
